Setup Guide for Academic Email Assistant — ALTERNATIVE (WSL) PATH
==================================================================

NOTE: Most users should follow README.md. README.md uses the one-click
PowerShell setup (`npm run setup`) which runs everything natively on Windows
with the OpenClaw CLI installed via npm.

This document describes a DIFFERENT, more manual setup that runs the
OpenClaw Gateway inside WSL2 (Ubuntu) and reaches Ollama on the Windows host
across the WSL network bridge. It is retained for advanced users and for
reference when troubleshooting clock drift, GPU/CPU placement, or context
window sizing issues.

The "Common issues" section at the bottom is useful regardless of which
setup path you used.

------------------------------------------------------------------

There are four pieces that need to be running:

* Ollama on Windows runs a local LLM on the GPU.
* OpenClaw Gateway runs in WSL and exposes a WebSocket API on port 18789.
* Webpack dev server serves the add-in over HTTPS at localhost:3000.
* The add-in itself is sideloaded into Outlook.

When all four are up, you can chat with an assistant inside Outlook.


Required software
-----------------

* Windows 10/11 with Hyper-V available (built in)
* WSL2 with Ubuntu 24.04: run "wsl --install -d Ubuntu-24.04" from PowerShell, then reboot
* Ollama: https://ollama.com/download/windows
* Node.js 22+ inside WSL via nvm: "curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash" then "nvm install 24"
* OpenClaw CLI inside WSL: "npm install -g openclaw@latest"
* A personal Outlook.com account. Custom add-in sideloading must be allowed. Most uni accounts block it.

Use a private or incognito browser window when sideloading. Old Microsoft sessions (work or uni accounts) can leak into Outlook.com and hide the custom add-in option.


One-time setup
--------------

1. Clone and install

   Inside WSL:

       cd ~ && mkdir -p dev && cd dev
       git clone https://github.com/nachtsheim/openclaw-outlook-addin.git
       cd openclaw-outlook-addin
       npm install
       npx office-addin-dev-certs install

   A Windows dialog will ask you to trust a root cert. Click Yes.


2. Pull a model and let Ollama listen on the network

   In Windows PowerShell:

       ollama pull qwen2.5:3b

   Then make Ollama reachable from WSL:

   a. Win key, type "environment variables", open "Edit the system environment variables".
   b. Under User variables, add OLLAMA_HOST set to 0.0.0.0:11434.
   c. Quit Ollama from the system tray and relaunch it.

   Open the firewall once, in PowerShell as Administrator:

       New-NetFirewallRule -DisplayName "Ollama WSL" -Direction Inbound -Protocol TCP -LocalPort 11434 -Action Allow


3. Onboard OpenClaw

   Inside WSL:

       openclaw onboard --install-daemon

   Wizard answers:

   * Setup mode: Manual, Local gateway
   * Model provider: Ollama
   * Ollama base URL: http://<WINDOWS-HOST-IP>:11434
     Find the IP with: ip route show | grep -i default | awk '{print $3}'
   * Model: pick "ollama/qwen2.5:3b" from the list
   * Gateway port: 18789
   * Gateway bind: Loopback
   * Gateway auth: Password.
   * Channels: skip them all
   * Hooks: Skip for now

   After onboarding, lower the model context window. The default is too large for an 8 GB GPU.

       nano ~/.openclaw/openclaw.json

   In nano, press Ctrl+W, type "contextWindow", hit Enter. Change 131072 to 16000. If there are two entries (one per model), change both. Save with Ctrl+O, Enter, then Ctrl+X.

   Restart the gateway:

       systemctl --user restart openclaw-gateway


4. Sideload the add-in into Outlook

   In a private or incognito window, sign into Outlook on the web:

   a. Open any email.
   b. Click "..." in the email header, then "Get Add-ins", then "My add-ins", scroll down to "Custom add-ins", then "+ Add a custom add-in", then "Add from file".
   c. Browse to \\wsl$\Ubuntu-24.04\home\<your-user>\dev\openclaw-outlook-addin\manifest.xml and install.



Daily startup
-------------

* Ollama: launch from the Start menu (or enable autostart in Windows Settings, Apps, Startup)
* OpenClaw Gateway: starts automatically as a systemd user service. Manual command: "systemctl --user start openclaw-gateway"
* Webpack dev server: does not auto-start. Run: "cd ~/dev/openclaw-outlook-addin && npm run dev"

The first time you open the sidebar, it'll ask for the gateway password (the one you picked during onboarding). After that it's saved in browser localStorage.

Common issues
-------------

"Unauthorized: gateway password missing" even though you entered the password

  This used to be a real problem: the gateway has two separate auth checks
  on the connect frame. One checks `auth.password`, the other checks
  `auth.token` for device signature verification. They don't fall through.

  As of this revision, the add-in already sends the token under both fields:

      auth: authToken ? { token: authToken, password: authToken } : {}

  See `src/taskpane/taskpane.js` in `sendHandshake()`. If you still see this
  error, check that `authToken` is being read correctly from localStorage
  (open the task pane DevTools, type `localStorage.getItem('acad-gateway-token')`)
  and that the value matches `gateway.auth.token` in `~/.openclaw/openclaw.json`.
 
"Disconnected" status that won't clear (other causes)

  * Webpack has stopped. Restart it with "npm run dev".
  * WSL clock drift causes "device signature expired" errors. Fix with "sudo ntpdate -s pool.ntp.org", or run "wsl --shutdown" from PowerShell and reopen WSL.

"model requires more system memory than is available"

  Context window is too large for VRAM. Drop it lower in ~/.openclaw/openclaw.json (try 8000 if 16000 still fails) or switch to a smaller model.

Ollama running on CPU instead of GPU

  Check with "ollama ps". The PROCESSOR column should say "100% GPU". If it isn't, end stale ollama.exe processes in Task Manager, update NVIDIA drivers, or drop the context window.


Useful paths
------------

* ~/.openclaw/openclaw.json: Gateway and model config
* ~/dev/openclaw-outlook-addin/src/taskpane/taskpane.js: Main add-in logic
* ~/dev/openclaw-outlook-addin/manifest.xml: Outlook manifest
* \\wsl$\Ubuntu-24.04\home\<user>\dev\openclaw-outlook-addin\: Repo, accessed from Windows File Explorer

Live gateway logs:

    journalctl --user -u openclaw-gateway -f
